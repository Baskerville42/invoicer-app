# Contributing to Electron Invoicer App

Дякуємо, що виявили інтерес до вдосконалення цього проекту! Як проект з відкритим кодом, ми вітаємо внески будь-якого роду, будь то виправлення помилок, нові функції або покращення документації.

## Як зробити внесок

### Повідомлення про помилки (Bug Reports)

Якщо ви знайшли помилку, будь ласка, створіть Issue, використовуючи шаблон Bug Report. Опишіть кроки для відтворення помилки та очікуваний результат.

### Пропозиції нових функцій (Feature Requests)

Ми завжди відкриті до нових ідей! Використовуйте шаблон Feature Request для опису вашої пропозиції.

### Pull Requests

1. Форкніть репозиторій.
2. Створіть нову гілку для вашої фічі (`git checkout -b feature/amazing-feature`).
3. Дотримуйтесь архітектурних принципів проекту (Feature-Based Structure).
4. Переконайтеся, що код відповідає стилю проекту та проходить лінтинг.
5. Запустіть тести перед відправкою.
6. Створіть Pull Request з детальним описом змін.

## Архітектурні принципи

Цей проект використовує **Feature-Based Architecture**. Це означає:

- Логіка згрупована за функціоналом (`src/features/`).
- Суворий поділ між `main` та `renderer` процесами.
- Використання TypeScript для максимальної типізації.
- Використання Tailwind CSS та FlyonUI для UI.

Детальніше про архітектуру читайте в `README.md`.

## Налаштування середовища

1. Клонуйте репозиторій.
2. Встановіть залежності: `npm install`.
3. Налаштуйте файл `.env` (див. `README.md`).
4. Запустіть додаток у режимі розробки: `npm run dev`.

## Кодекс поведінки

Беручи участь у цьому проекті, ви погоджуєтеся дотримуватися нашого [Кодексу поведінки](CODE_OF_CONDUCT.md).
